Day 1 Foundations — Building Trust Through Immediate Value

Lesson 1: Foundations & Setup
Day 1 • 9:00 AM
1hr 20min
20 min Lecture
  • Why regex, git, and error messages are perfect entry points
  • The psychology of "low stakes" AI interactions
  • Tool landscape: ChatGPT vs Claude vs Copilot vs local models
  • Understanding output variance — AI is probabilistic, not deterministic
  • Live demo: Same prompt, different outputs across tools
60 min Exercise (includes breaks)
  • Setup: Everyone creates accounts on 2+ AI tools
  • Task 1: Generate 5 regex patterns you've struggled with before
  • Task 2: Use AI to explain a cryptic git reflog situation
  • Task 3: Feed it your worst error message from the past month
  • Task 4: Run same prompt on different tools — compare results
  • Discussion: Share your best output with the group
Starter Prompt
"Write a regex that matches [your specific case]. Explain each part of the pattern and give me 5 test cases that should match and 5 that shouldn't."
Outcome: 4+ concrete wins to use tomorrow + understanding of tool differences
Lesson 2: Code Explanation & Learning
Day 1 • 10:30 AM
1hr 20min
20 min Lecture
  • The safest AI use case: asking questions costs nothing
  • "What does this do?" — the zero-risk starting point
  • Reading unfamiliar languages, frameworks, and patterns
  • Demo: Explaining a gnarly piece of code in 60 seconds
60 min Exercise (includes breaks)
  • Task 1: Find code you've never understood — get it explained
  • Task 2: Paste code in a language you don't know — learn what it does
  • Task 3: Ask "why was it written this way?" on a confusing pattern
  • Task 4: Get a line-by-line breakdown of a complex function
Starter Prompt
"Explain this code like I'm a developer who's never seen this language/framework before. What does it do, and why might someone write it this way?"
Outcome: Confidence to explore any codebase without fear
Lesson 3: Context Engineering
Day 1 • 11:50 AM
1hr 10min
20 min Lecture
  • Why prompt quality determines output quality
  • Core patterns: role, context, task, format, examples
  • Iteration strategies: refining prompts systematically
  • The "prompt library" concept
50 min Exercise (includes breaks)
  • Task 1: Transform a vague prompt into a precise one
  • Task 2: Use few-shot examples to control output format
  • Task 3: Create a reusable prompt template
  • Group work: Build a shared prompt library for common tasks
Template Pattern
"You are a [role]. Given [context], your task is to [specific action]. Output should be [format]. Here's an example: [example]."
Outcome: Personal prompt library with 10+ reusable templates

Day 2 Frameworks & Constraints — When Structure Helps (and Hurts)

Lesson 4: Working Within the Framework
Day 2 • 9:00 AM
1hr 20min
20 min Lecture
  • Why frameworks are AI's comfort zone
  • Game engines (Godot, Unity, Unreal) as structured environments
  • How AI leverages patterns, conventions, and documentation
  • Demo: Building a game mechanic in Godot with AI
60 min Exercise (includes breaks)
  • Task 1: Generate a player controller script in Godot/Unity
  • Task 2: Ask AI to implement a game mechanic (inventory, combat, etc.)
  • Task 3: Debug framework-specific errors with AI assistance
  • Task 4: Compare output quality when you specify the framework vs. generic request
Framework-Aware Prompt
"In Godot 4 using GDScript, create a 2D player controller with: WASD movement, jump with coyote time, and animation state machine. Use CharacterBody2D."
Outcome: Understanding that framework context dramatically improves AI output
Lesson 5: Overconstraining the AI — Overconstraining the AI
Day 2 • 10:30 AM
1hr 20min
20 min Lecture
  • The constraint spectrum: too little vs. too much guidance
  • Case study: React/Vue website vs. vanilla HTML/CSS/JS
  • When frameworks help AI (patterns, conventions, examples)
  • When frameworks hurt AI (outdated training, niche APIs, version conflicts)
60 min Exercise (includes breaks)
  • Task 1: Build the same component in React AND vanilla JS — compare results
  • Task 2: Intentionally overconstrain: specify framework + version + obscure library
  • Task 3: Intentionally underconstrain: "build me a website" with no guidance
  • Discussion: When did constraints help? When did they create problems?
Experiment Prompts
Overconstrained: "Using React 18.3 with Zustand 4.5, React Query 5, and Radix UI, build a..."

Underconstrained: "Build me a todo app"

Balanced: "Build a todo app in React with state management. I'm flexible on libraries."
Outcome: Intuition for when to specify frameworks vs. let AI choose
Lesson 6: Generating Readable Code
Day 2 • 11:50 AM
1hr 10min
20 min Lecture
  • The "trust but verify" mindset — you MUST read the code
  • Why type annotations matter for AI-generated code
  • Testing as a verification mechanism
  • Demo: CLI tool with full type hints, tests, and documentation
50 min Exercise (includes breaks)
  • Project: Build a CLI tool from scratch with AI
  • Requirements: argparse/click, type annotations, pytest tests, docstrings
  • Task 1: Generate the initial structure
  • Task 2: Read every line — find at least 2 things you'd change
  • Task 3: Run the tests — do they actually test the right things?
  • Task 4: Refactor one function based on your review
Project Prompt
"Create a CLI tool that [your choice: converts files, fetches data, generates reports, etc.]. Include: click for CLI, full type annotations, pytest tests with >80% coverage, and Google-style docstrings. Structure as a proper package with pyproject.toml."
Outcome: Habit of examining AI code critically + a working CLI tool to take home
